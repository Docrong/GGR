任务调度组件使用方法：
 
  组件功能描述：实现任务的定制；自动按照计划执行任务；实时监控任务的执行；及时报警超时执行的任务；
  
  使用：
  1、任务类型的配置：通过页面化的方式，用户可以配置系统可以支持的任务执行类型。任务类型的配置信息
包括：任务类型名称、任务类型实现类的类名和该类任务允许执行的最长时间。
     其中, 任务类型实现类：通过该类对象的execute方法，实现任务的完成。
           任务执行最长时间：若任务执行时间大于此时限的话，系统会及时提醒系统管理员关注此任务的执行情况。
     若要新增一类任务，需要系统管理员编写该类任务的实现类，然后配置该任务类型即可。
     
  2、任务的定制：根据系统提供的任务类型，用户可以定制任务。任务的定制也是通过页面方式实现的。
 在任务定制页面上，用户可以选择该任务所属的任务类型、任务执行类型（simple or cron）以及执行周期
 等信息。需要说明的是：
     若执行类型为simple ，则执行周期是以毫秒为单位的数值，表明任务执行的间隔时间。例如：我们
 填写300000，则表示该任务每隔5分钟执行一次。
     若执行类型为cron，则执行周期的填写应遵循crontab时间格式，表明该任务执行时间。例如：
 我们填写0 0 10 * * ?，则表示该任务每天的10点钟执行一次。
